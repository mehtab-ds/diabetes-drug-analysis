# 🧪 Clinical Trial Data Cleaning & Exploratory Analysis  
*Turning raw clinical trial data into actionable healthcare insights*

## 📌 Project Overview
This project demonstrates a complete workflow for handling clinical trial data.  
Multiple raw datasets (patients, treatments, adverse reactions) are cleaned, merged, and analyzed to uncover insights into treatment effectiveness and safety.  

The focus is on **data engineering** (cleaning, merging, structuring) and **analytical reasoning** (EDA, visualization, interpretation).

---

## 🔧 Tech Stack
- Python  
- Pandas  
- NumPy  
- Matplotlib / Seaborn  
- Jupyter Notebook  

---

## 📂 Repository Structure
- **data/**  
  - `treatments.csv` → Raw treatment data  
  - `treatments_cut.csv` → Trimmed treatment dataset  
  - `patients.csv` → Patient demographics and details  
  - `adverse_reactions.csv` → Reported adverse reactions  
  - `final_merged_clinical_trial_data.csv` → Cleaned and merged dataset for analysis  

- **notebooks/**  
  - `data_cleaning.ipynb` → Cleaning and merging workflow  
  - `EDA_Clinical_Trial.ipynb` → Exploratory analysis and visualizations  

---

## ⚙️ Workflow
1. **Data Cleaning**  
   - Handle missing values and inconsistencies  
   - Standardize column names and formats  
   - Remove duplicates  

2. **Data Merging**  
   - Integrate patients, treatments, and adverse reactions  
   - Produce a consolidated dataset for reproducible analysis  

3. **Exploratory Data Analysis (EDA)**  
   - Generate descriptive statistics  
   - Visualize treatment outcomes and adverse reaction trends  
   - Identify demographic patterns (age, BMI, etc.)  

---

## 📈 Key Insights
- **Treatment comparisons** reveal differences in adverse reaction frequency.  
- **Patient demographics** (age, BMI) influence treatment outcomes.  
- **Merged dataset** enables reproducible and transparent analysis across multiple trial dimensions.
